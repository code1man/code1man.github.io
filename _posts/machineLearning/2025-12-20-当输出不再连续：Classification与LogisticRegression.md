# 分类问题并不神秘，本质是：如何用连续函数表示离散决策

在回归问题中，我们预测的是一个连续数值；而在分类问题中，我们关心的是决策本身。本节的核心问题只有一个：

**如何用连续可优化的函数，做出离散的分类决策？**

## 1. 从回归继承而来：连续 vs 离散

- **回归问题**：模型输出是连续值，例如房价、温度、分数。
- **分类问题**：模型输出是离散标签，例如 {猫, 狗, 鸟}。

乍一看，分类和回归似乎完全不同，但它们的差别并不在于模型结构，而在于输出的表示方式。

**一个关键陷阱：用数字表示类别**

假设我们直接用数字表示类别：

- A = 0
- B = 1
- C = 2

这会隐含一个错误的假设：C > B > A，且 B 在“数值上”比 A 更接近 C。但在分类语义中，类别之间没有大小关系，也不存在“邻接关系”。数值上的邻接，并不代表语义上的相似。

## 2. 独热编码（One-Hot Encoding）——分类的第一道门槛

为了解决“虚假的数值关系”，我们引入 One-Hot Encoding。

- 用一个向量表示类别，每一维对应一个类别。

以 3 类为例：

| 类别 | One-hot   |
| ---- | --------- |
| A    | [1, 0, 0] |
| B    | [0, 1, 0] |
| C    | [0, 0, 1] |

**为什么这样做是对的？**

- 不引入大小关系
- 类别之间正交
- 任意两个不同类别的欧氏距离恒为 2

这使得模型不会“误以为”某些类别更接近。

**代码示例**：

```python
import numpy as np
labels = np.array([0, 2, 1])
one_hot = np.eye(3)[labels]
```

><font color=camel>一个重要代价</font>：One-hot 编码会直接变成特征，类别数 = 特征维度。类别很多时特征维度会爆炸，这也是 NLP 中常用 embedding 的原因。

## 3. “以前算一次，现在算 n 次”——从回归到多分类

在回归中，我们只有一个输出：

$$
y = w^T x + b
$$

而在多分类中，我们需要为每一个类别都算一次分数：

$$
z_k = w_k^T x + b_k, \quad k=1,2,...,K
$$

矩阵形式：

$$
z = Wx + b
$$

这一步非常关键，因为这正是从 Logistic Regression 走向 Neural Network 的桥梁。输出不再是一个标量，而是一个向量。

## 4. Softmax：把“分数”变成“概率”

线性模型输出的只是 score（分数），但分类问题需要的是概率分布。Softmax 的作用正是：

- 把分数变成正数
- 归一化为和为 1 的概率

数学形式：

$$
\text{softmax}(z_i) = \frac{e^{z_i}}{\sum_j e^{z_j}}
$$

**数值稳定的实现**：

```python
def softmax(z):
  z = z - np.max(z)   # 防止 overflow
  exp_z = np.exp(z)
  return exp_z / np.sum(exp_z)
```

减去 max 不会改变结果，但可以显著提升数值稳定性。

## 5. Loss Function：为什么 MSE 不好，Cross-Entropy 好？

### 5.1 MSE 为什么不适合分类？

- 概率假设不合理：MSE 隐含高斯噪声假设，但分类本质是离散事件。
- 梯度问题：在 sigmoid / softmax 下，MSE 会导致梯度变小，训练缓慢。
- 对错误分类不敏感：预测错了类别，惩罚不够“狠”。

### 5.2 Cross Entropy 的优势

交叉熵可以从概率角度理解为最大化正确类别的对数概率。多分类的形式为：

$$
L = -\sum_k y_k \log(\hat{y}_k)
$$

其中：

- $y_k$：one-hot 标签
- $\hat{y}_k$：softmax 输出概率

当模型对正确类别信心很低时，$\log(\hat{y}) \to -\infty$，惩罚非常大，梯度清晰，学习效率高。

## 6. 逻辑回归的局限性：问题不在模型，而在特征

逻辑回归的决策边界本质上是线性的。如果在原始特征空间中数据线性不可分，那么逻辑回归一定失败 —— 无论你怎么调参、怎么优化。

**特征转换能解决吗？**

可以，但特征转换通常并不简单，需要知道用什么非线性、变成多少维、哪些组合有意义，这些往往依赖领域知识，而不是模型自动学会的。

## 7. 从特征工程到深度学习：级联逻辑回归

一个自然的想法是让模型自己学特征。方法是把多个逻辑回归模型级联起来：

$$
x \rightarrow \text{Logistic} \rightarrow \text{Logistic} \rightarrow \text{Logistic}
$$

每一层都在做线性变换，加一次非线性（sigmoid / ReLU），这实际上就是神经网络。

**一个关键变化：联合学习（Joint Learning）**

- 特征不再是人工设计
- 分类器不再是单独训练
- 所有参数通过同一个 loss 联合优化

这正是深度学习“端到端学习”的核心思想。

---

**总结**：逻辑回归并不弱，它弱在对特征的依赖。当数据在当前特征空间中不可分时，逻辑回归一定失败；而深度学习的核心思想，正是通过级联多个逻辑回归模型，让模型自动学习合适的特征表示，并通过联合优化所有参数，实现端到端的分类决策。

> 分类问题的本质，是用连续函数优雅地表达离散决策。
