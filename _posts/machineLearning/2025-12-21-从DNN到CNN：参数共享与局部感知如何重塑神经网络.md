## 1. DNN / FC / CNN：自然演化路线

模型的能力，核心在于特征表示，而不是损失函数或优化器本身。逻辑回归、多分类模型本质上是线性模型加非线性映射。DNN（全连接网络）让模型可以自动学习特征变换，但面对图像、语音、棋盘等结构化数据时，全连接网络存在参数太多、忽略数据结构、学习效率低等问题，这正是卷积神经网络（CNN）登场的原因。

### 1.1 全连接网络（FC / DNN）
形式：$y = f(Wx + b)$  
特点：每个神经元“看整个输入”，不关心空间结构，参数量随输入维度线性增长。适合表格数据，不适合高维结构化数据。

### 1.2 为什么 FC 不适合图像 / 语音 
以 $224 \times 224 \times 3$ 图片为例，展平成向量后维度为 150,528，第一层 FC（1000 个神经元）参数量约 $1.5 \times 10^8$，导致参数爆炸、相邻像素关系被破坏、平移后需重新学习。

### 1.3 CNN 

#### 1.3.1 CNN为什么有效
**解释一：多个神经元盯着不同局部区域（局部感知与参数共享）**

- 在卷积神经网络（CNN）中，每个神经元只关注输入数据的一个小局部区域（称为感受野或 patch），而不是全局。
- 这些局部区域可以相互重叠，确保特征的连续性和空间信息的完整捕捉。
- 由于自然图像等结构化数据在不同空间位置往往具有相似的统计特性，CNN 假设“同一种特征在不同位置都可能出现”，因此可以在所有位置共享同一组卷积核参数。
- 这种参数共享机制极大地减少了模型参数数量，提高了训练效率，并赋予模型平移等变性（即特征无论出现在何处都能被检测到）。

**解释二：一个神经元扫描整张图片（特征检测器视角）**

- 可以将卷积核视为一个特征检测器，它在输入图像上滑动（卷积操作），对每个局部区域进行相同的特征提取。
- 每当卷积核检测到目标特征（如边缘、角点等）时，对应的神经元就会被激活，无论该特征出现在图像的哪个位置。
- 这种机制使得 CNN 更关注“是否存在某种模式”，而不是“该模式具体出现在哪里”，从而提升了模型对空间位置变化的鲁棒性。
- 通过多层堆叠，CNN 能够逐步从简单的局部特征组合出更复杂的全局表达，实现对高维结构化数据的高效建模。 

#### 1.3.2 CNN 的应用场景
卷积本质是一个降维 + 降参数的假设模型。适用前提：
1. 不需要一开始看全局
2. 同样的判断逻辑会反复出现
3. FC 参数量不可接受

典型任务：图像、语音、视频、棋盘、局部上下文强的 NLP（字符级、CNN-based NLP）

### 1.3.3 CNN 流程

卷积 → 池化 →（重复）→ 全连接 / 全局池化  
- 卷积：提取局部特征  
- 池化：降维 + 增强鲁棒性  
- FC：最终决策（有时可省略）


## 2. 卷积操作：CNN 的核心

### 2.1 卷积的数学形式（二维）

设输入特征图为 $X$，卷积核为 $K$：  
$$
Y(i, j) = \sum_{m, n} X(i+m, j+n) K(m, n)
$$  
多通道输入（RGB）：  
$$
Y = \sum_{c=1}^C X_c * K_c
$$

### 2.2 卷积核与参数共享

卷积核大小固定（如 $3 \times 3$），在整张图上共享同一组参数。参数量 $= k \times k \times C \times N$，与图片尺寸无关，是 CNN 比 FC 参数少的根本原因。

> 为什么经典卷积核是 $3 \times 3$？
    >- 单层 $3 \times 3$ 感受野小，多层叠加后感受野迅速扩大（两层 $3 \times 3$ → $5 \times 5$，三层 $3 \times 3$ → $7 \times 7$）。
    >- 参数少，非线性更多，表达能力更强。
    >- 多层小卷积 ≈ 一层大卷积，但更强。

## 3. CNN超参数

### 3.1 Padding（补零）

防止尺寸快速变小，保留边缘信息。  
输出尺寸：$\text{Out} = N + 2P - K / S + 1$

### 3.2 Stride（步长）

控制卷积移动速度，步长越大降维越快。

### 3.3 Overlap

感受野重叠，保证特征连续性，防止“块状感知”。

## 4. 池化（Pooling）：降维但不学参数

#### 4.1 常见池化

Max Pooling、Average Pooling  
数学形式（Max Pool）：$y = \max(x_1, x_2, ..., x_n)$

#### 4.2 作用

降低空间分辨率，提高平移不变性，减少计算量。

#### 4.3 尺寸计算示例

输入 $32 \times 32$，Kernel $2 \times 2$，Stride=2，输出 $16 \times 16$。

#### 4.4 全局池化

$H \times W \times C \rightarrow 1 \times 1 \times C$，常见于 ResNet，参数更少，防止过拟合。

#### 4.5 卷积与池化的代码示例（PyTorch）

```python
import torch
import torch.nn as nn

conv = nn.Conv2d(
    in_channels=3,
    out_channels=16,
    kernel_size=3,
    stride=1,
    padding=1
)

pool = nn.MaxPool2d(kernel_size=2, stride=2)

x = torch.randn(1, 3, 32, 32)
y = pool(conv(x))
print(y.shape)
```

**总结**  
FC：什么都看，但代价巨大  
CNN：只看局部，但重复判断  
局部 + 共享 + 层叠 = 高效表达  
> CNN 不是更聪明，而是把正确的先验假设写进了模型结构里。
