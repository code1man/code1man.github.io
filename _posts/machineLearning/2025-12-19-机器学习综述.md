## 1. AI、机器学习、深度学习关系

- **人工智能（AI, Artificial Intelligence）**：广义上指让计算机具备“智能”能力的所有技术，包括规则系统、专家系统、搜索算法等。
- **机器学习（Machine Learning）**：AI 的一个子领域，核心思想是让计算机通过数据自动学习规律，无需显式编程。机器学习包括监督学习、非监督学习、强化学习等多种方法。
- **深度学习（Deep Learning）**：机器学习的一个分支，主要指基于多层神经网络的学习方法。深度学习通过多层结构自动提取特征，适用于图像、语音、自然语言等复杂任务。

- **层级关系**：AI ⊃ 机器学习 ⊃ 深度学习

> 📌 一句话总结：  
> 深度学习是机器学习的一种实现方式，而机器学习是实现 AI 的一种路径。

## 2. 机器学习的本质

- 机器学习的本质是**通过数据学习一个函数**，使得输入数据后能输出期望的结果。
- 换句话说，机器学习就是**自动拟合一个未知的函数**，让模型能够完成预测、分类等任务。
- 形式化表达：给定输入 $x$，学习一个函数 $f(x; w)$，其中 $w$ 是模型参数，使得 $f(x; w)$ 尽可能接近真实输出 $y$。
- 机器学习的目标是**最小化预测值与真实值之间的误差**，不断优化参数 $w$，提升模型的泛化能力。

- 实际应用举例：
  - 图像识别：输入图片，输出类别标签。
  - 房价预测：输入房屋特征，输出价格。
  - 语音识别：输入音频信号，输出文本内容。

- 总结：机器学习就是让计算机通过数据自动“学会”一个函数，完成各种智能任务。

## 3. 机器学习函数类型

- **回归（Regression）**：输出为连续数值。  
  - 示例：房价预测、温度预测、股票价格预测等。
- **分类（Classification）**：输出为离散类别。  
  - 示例：垃圾邮件识别（是否为垃圾邮件）、图像分类（猫/狗/鸟）、疾病诊断（阳性/阴性）等。
- **聚类（Clustering）**：将数据自动分组，每组称为一个簇，类别标签未知。  
  - 示例：客户分群、图像分割、市场细分等。
- **异常检测（Anomaly Detection）**：识别与大多数数据不同的异常样本。  
  - 示例：信用卡欺诈检测、设备故障预警等。
- **排序（Ranking）**：对对象进行排序，常用于推荐系统和搜索引擎。  
  - 示例：搜索结果排序、商品推荐排序等。
- **生成（Generation）**：根据输入生成新的数据。  
  - 示例：文本生成、图像生成（如GAN）、语音合成等。
- **结构化输出（Structured Prediction）**：输出为复杂结构，如序列、树或图。  
  - 示例：序列标注（分词、命名实体识别）、图像分割、机器翻译等。

> PS：结构化输出与普通分类/回归的区别在于，结构化输出的目标是预测具有内部依赖关系的<font color=tomato>复杂结构（如句子的词性序列、图像的像素标签等）</font>，而普通分类/回归只需预测单一标签或数值。结构化输出需要考虑输出各部分之间的关联性，建模难度更高。

## 4. 机器学习的基本步骤

### 4.1 定义含有未知参数的函数

- 例如：$y = f(x; w; b)$，$w$、$b$为待学习参数。

### 4.2 定义损失函数

#### 4.2.1 损失函数类别
- 损失函数衡量模型预测值与真实值之间的差距，是模型优化的目标。
- 常见损失函数及其适用场景：
  - **平均绝对误差（MAE）**：适用于回归任务，尤其当对异常值不敏感时。
    ```math
    \mathrm{MAE} = \frac{1}{n} \sum_{i=1}^n |y_i - \hat{y}_i|
    ```
    适用场景：房价预测、温度预测等，对异常值鲁棒性要求较高的回归问题。
  - **均方误差（MSE）**：常用于回归任务，对异常值更敏感。
    ```math
    \mathrm{MSE} = \frac{1}{n} \sum_{i=1}^n (y_i - \hat{y}_i)^2
    ```
    适用场景：图像重建、信号还原等，对大误差惩罚更强的回归问题。
  - **交叉熵损失（Cross Entropy）**：用于分类任务，衡量预测概率分布与真实分布的差异。
    ```math
    \mathrm{CrossEntropy} = -\sum_{i=1}^n \sum_{k=1}^K y_{ik} \log(\hat{y}_{ik} + \epsilon)
    ```
    其中 $K$ 为类别数，$y_{ik}$ 为样本 $i$ 的真实标签（one-hot 编码），$\hat{y}_{ik}$ 为模型对类别 $k$ 的预测概率，$\epsilon$ 是一个很小的常数（如 $1 \times 10^{-8}$），用于避免 $\log(0)$ 错误。
    适用场景：图像分类、文本分类等多类别分类问题。
- 损失函数的选择需结合具体任务和数据特点，合理选择有助于提升模型性能。
- 损失函数的作用是指导参数优化，使模型预测更准确。
#### 4.2.2 损失函数代码实现

下面以 Python（NumPy）为例，分别实现 MSE、MAE 和交叉熵损失函数的 `compute_loss()`：

```python
import numpy as np

def compute_loss(y_true, y_pred, loss_type='mse'):
  """
  y_true: 真实标签（回归为一维，分类为one-hot编码）
  y_pred: 预测值（回归为一维，分类为概率分布）
  loss_type: 'mse', 'mae', 'cross_entropy'
  """
  if loss_type == 'mse':
    # 均方误差
    return np.mean((y_true - y_pred) ** 2)
  elif loss_type == 'mae':
    # 平均绝对误差
    return np.mean(np.abs(y_true - y_pred))
  elif loss_type == 'cross_entropy':
    # 交叉熵损失，适用于多分类，y_true为one-hot
    epsilon = 1e-8
    y_pred = np.clip(y_pred, epsilon, 1 - epsilon)
    return -np.sum(y_true * np.log(y_pred)) / y_true.shape[0]
  else:
    raise ValueError("Unsupported loss_type")
```

**用法示例：**

```python
# 回归任务
y_true = np.array([3.0, 2.5, 4.1])
y_pred = np.array([2.8, 2.7, 4.0])
print(compute_loss(y_true, y_pred, loss_type='mse'))  # MSE
print(compute_loss(y_true, y_pred, loss_type='mae'))  # MAE

# 分类任务（one-hot编码）
y_true = np.array([[1,0,0],[0,1,0],[0,0,1]])
y_pred = np.array([[0.8,0.1,0.1],[0.2,0.7,0.1],[0.1,0.2,0.7]])
print(compute_loss(y_true, y_pred, loss_type='cross_entropy'))  # Cross Entropy
```
### 4.3 梯度下降

- 通过计算损失函数对参数的梯度，逐步优化参数。
- 超参数：学习率、迭代次数、batchsize等。
- 数学依据：沿梯度方向减少损失。（其实并不准确）

- 代码示例：

  **PyTorch 示例**：
  ```python
  import torch

  w = torch.randn(1, requires_grad=True)
  optimizer = torch.optim.SGD([w], lr=0.01)

  for epoch in range(num_epochs):
      loss = compute_loss(w)  
      optimizer.zero_grad()
      loss.backward()
      optimizer.step()
  ```

  **NumPy 示例**：
  ```python
  import numpy as np

  w = np.random.randn(1)
  learning_rate = 0.01

  for epoch in range(num_epochs):
      grad = compute_gradient(loss, w)  
      w = w - learning_rate * grad
  ```

  **TensorFlow 示例**：
  ```python
  import tensorflow as tf

  w = tf.Variable(tf.random.normal([1]), dtype=tf.float32)
  optimizer = tf.keras.optimizers.SGD(learning_rate=0.01)

  for epoch in range(num_epochs):
      with tf.GradientTape() as tape:
          loss = compute_loss(w) 
      grad = tape.gradient(loss, [w])
      optimizer.apply_gradients(zip(grad, [w]))
  ```

### 4.4 为什么梯度下降能行
教材经常会说：“每次更新参数，都会得到一个使得 $L(\theta)$ 更小的 $\theta$。”

❌ **常见但不严谨的说法**  
梯度下降每次都会让 loss 变小。

✅ **更准确的说法**  
在**足够小的学习率**下，沿着负梯度方向更新参数，在**局部一阶近似**意义下，可以保证 loss 下降。

> 📌 关键词：足够小、局部、一阶近似

---

#### 4.4.1 为什么“负梯度方向”能让 Loss 变小？

##### 1、理论依据：一阶泰勒展开

对一个可微函数 $L(\theta)$，在当前点 $\theta$ 附近做一阶泰勒展开：

$$
L(\theta + \Delta\theta) \approx L(\theta) + \nabla L(\theta)^T \Delta\theta
$$

这是**局部线性近似**。

##### 2️、梯度下降的推导

梯度下降的更新为：

$$
\Delta\theta = -\eta \nabla L(\theta)
$$

代入泰勒展开：

$$
L(\theta - \eta \nabla L) \approx L(\theta) - \eta \|\nabla L(\theta)\|^2
$$

只要 $\eta > 0$ 且学习率足够小，忽略高阶项是合理的，右边一定小于 $L(\theta)$。

> **意义**：负梯度方向是局部一阶近似下，下降最快的方向。

---

##### 3、现实与理论的差距

实际中，loss 不一定每次都下降，原因包括：

- **忽略了二阶及以上项**  
  完整泰勒展开为：
  $$
  L(\theta + \Delta\theta) = L(\theta) + \nabla L(\theta)^T \Delta\theta + \frac{1}{2} \Delta\theta^T H \Delta\theta + \dots
  $$
  - 学习率太大 → 二阶项主导 → loss 可能上升
  - 非凸函数 → 局部近似失效
  - Mini-batch SGD 带噪声，loss 有波动

> 📌 这也是为什么要调学习率、会出现震荡、Adam/Momentum 等优化方法被提出。

---

##### 4、泰勒展开的作用

> **泰勒展开不是用来证明“梯度下降一定有效”，而是解释为什么选择负梯度作为更新方向。**

| 作用     | 说明                             |
| -------- | -------------------------------- |
| 理论动机 | 负梯度 = 局部最陡下降方向        |
| 条件说明 | 只在小步长、局部有效             |
| 铺垫后续 | 二阶方法、优化算法改进的理论基础 |

## 5. 提升模型学习能力的方法

- **提升特征维度**：通过引入更多的输入特征（如多种传感器数据、组合特征、特征工程等），让模型能够捕捉到更丰富的信息，从而提升模型的表达能力和拟合能力。但需要注意，盲目增加特征可能导致“维度灾难”，需结合特征选择与降维方法。
- **增加模型参数数量**：通过增加模型的参数（如神经网络中的神经元数量、层数，或决策树的深度等），提升模型的复杂度，使其能够拟合更复杂的函数关系。但参数过多可能导致过拟合，需要配合正则化等手段。
- **提升模型的非线性表达能力**：引入非线性激活函数（如Sigmoid、ReLU、Tanh等），使模型能够拟合非线性关系，突破线性模型的局限性。非线性激活函数是深度学习模型强大表达能力的关键。
- **集成学习方法**：如Bagging、Boosting、Stacking等，通过组合多个模型提升整体性能，降低单一模型的偏差和方差。
- **正则化与归一化**：通过L1/L2正则化、Dropout、Batch Normalization等技术，提升模型泛化能力，防止过拟合。
- **数据增强与扩充**：通过对训练数据进行扩充（如图像旋转、裁剪、噪声扰动等），提升模型对不同场景的适应能力。

> 总结：提升模型学习能力的方法包括增加特征、提升模型复杂度、引入非线性、采用集成与正则化等手段，需根据实际任务和数据情况灵活选择与组合。

## 6. 激活函数与神经元

神经网络的核心在于**激活函数**和**神经元结构**。激活函数赋予模型非线性表达能力，是深度学习的关键。

### 6.1 Sigmoid函数

- **定义**：$f(x) = \frac{1}{1 + e^{-x}}$
- **特性**：
  - 输出范围在 $(0, 1)$，适合概率建模。
  - 参数 $w$ 控制斜率，$b$ 控制偏移，$c$ 控制输出范围。
  - 多特征时，形式为 $f(x) = sigmoid(w_1x_1 + w_2x_2 + ... + b)$。
- **应用**：常用于二分类任务。
- **变种**：hard sigmoid 是简化版，计算更快。

### 6.2 ReLU函数

- **定义**：$f(x) = \max(0, x)$
- **特性**：
  - 非线性，计算简单，收敛快。
  - 有效缓解梯度消失问题。
- **应用**：深度网络中广泛使用。

### 6.3 为什么要用激活函数？

- **线性模型限制**：没有激活函数时，神经网络本质是线性变换，表达能力有限。
- **非线性能力提升**：激活函数引入非线性，使模型能拟合复杂函数。
- **抑制噪声与上限**：如sigmoid可抑制极端值，ReLU可抑制负值影响。
- **防止分散**：激活函数有助于稳定训练过程。

### 6.4 神经元结构

- 一个神经元的输出：$y = f(w^Tx + b)$
- $f$ 为激活函数，$w$ 为权重，$b$ 为偏置（bias）。

### 6.5 关于 bias（偏置）的工程实现与直觉

#### 6.5.1 原理
- **做法**：给<font color=plum>特征向量 x </font>增加一维，其值恒为 1，对应权重就是 bias。
- 数学表达：
  $$
  w^Tx + b \equiv \tilde{w}^T\tilde{x}
  $$
  其中
  $$
  \tilde{x} = [1, x_1, x_2, ..., x_k]^T,\quad \tilde{w} = [b, w_1, w_2, ..., w_k]^T
  $$

#### 6.5.2 原因

- $w^Tx + b = 0$ 表示一个超平面。
- 没有 bias 时，决策边界必须经过原点。
- 有 bias 时，决策边界可以整体平移，模型有“平移自由度”。

#### 6.5.3 数学与工程实现

1️⃣ **方式一：显式 bias（最常见）**  

$y = w @ x + b$  
- 代码清晰，易于理解  
- 主流数值库/框架（如 PyTorch、TensorFlow）默认采用此方式

2️⃣ **方式二：合并 bias（教学 & 数学常用）**  
将 bias 合并到特征和权重中：
```python
# x: 原始特征矩阵，形状 (N, k)
x = np.concatenate([np.ones((N, 1)), X], axis=1)  # 第一维恒为 1
w = np.zeros(k + 1)  # 权重向量，第一维对应 bias
y = x @ w
```
- 第一维恒为 1，对应的权重就是 bias
- 这个权重和其他 $w$ 一样被梯度下降更新，不是固定不动的

> 总结：两种方式本质等价，选择哪种取决于代码习惯和具体场景。

#### 6.5.4 常见误区

- bias 并不是“把 $x$ 变成 $x+1$”，而是增加一维恒为 1 的特征。
- 这样做不会改变模型本质，只是方便实现和表达。

## 7. 梯度下降优化

- **Batch（批量）**：在训练过程中，通常不会每次都用全部训练数据（即全量梯度下降），而是将数据集分成若干小批次（batch），每次只用一个batch来更新参数。这样做的原因有：
  - **内存效率**：一次处理全部数据对内存要求极高，尤其是大规模数据集时难以实现。
  - **训练速度**：小批量数据可以更快地完成一次参数更新，提高训练效率。
  - **泛化能力**：每个batch只包含部分数据，引入了噪声，有助于跳出局部最优，提升模型泛化能力。
  - **并行计算**：batch中的样本可以在 GPU 等硬件上并行处理，大幅提升计算速度。
    >  <font color=orange>PS: 并行计算之所以可行，是因为 batch 中的每个样本在前向传播和反向传播时相互独立，没有依赖关系。因此，这些样本可以同时在 GPU 等支持并行运算的硬件上处理，从而显著提升整体计算效率和训练速度。</font>
   
  - **为什么不一次看完所有训练集**：全量梯度下降（Batch Gradient Descent）虽然理论上最精确，但计算量大、速度慢，且对硬件要求高。小批量（Mini-batch）方法在效率和效果之间取得了平衡。

- **Epoch（轮次）**：指所有训练数据被完整训练一次。通常需要多个epoch，模型才能充分学习数据特征。

  - **普遍性定理**：理论上，任意连续函数都可以用一个隐藏层的神经网络（浅层网络）来逼近，即浅层网络具备“万能近似”能力。但在实际应用中，深层网络（多层结构）往往更加高效和易于训练。

  - **深度=模块化**：深度网络可以看作是自动化的“模块化”结构。类似于编程时将复杂任务拆分为多个函数模块，深度网络通过多层结构分层抽象特征，每一层负责不同层次的信息处理。这种分层结构有助于模型自动学习数据的高阶特征，提升泛化能力。

    - **举例说明**：比如要识别“长头发男”“长头发女”“短头发男”“短头发女”，如果用浅层网络，需要为每种组合都准备足够的样本，且样本分布可能极不均衡。而深度网络可以将“头发长短”与“性别”分别作为不同层次的特征，组合学习，极大缓解样本不均衡问题，提高数据利用效率。

  - **理论与实验支持**：Bengio等人在《Deep Learning》（2016）等研究中指出，增加网络深度可以更高效地表达复杂函数，深层结构能够用较少参数实现同样的表达能力，而“矮胖”结构（浅层但宽）往往需要指数级参数才能达到类似效果。这也是深度神经网络成为主流的原因之一。

  - **模型压缩与知识蒸馏**：论文《Do Deep Nets Really Need to Be Deep?》提出了“Teacher-Student”模型压缩（知识蒸馏）方法：先训练一个深度网络（Teacher），再用其输出（soft labels）指导浅层网络（Student）训练。结果发现，浅层网络如果通过模仿深度网络的行为，也能获得接近深度网络的性能。这说明“深度”并非性能提升的唯一决定性因素。

    - **深度的真正价值**：虽然浅层网络理论上可以拟合复杂函数，但实际训练时很难直接从原始标签获得优秀性能。深度网络的优势在于其结构更易于优化，训练算法（如梯度下降）更容易找到优秀解。因此，深度网络的实际价值更多体现在优化可行性和训练效率上，而不仅仅是表达能力。

    > 总结：浅层网络理论上可以拟合任意函数，但深度网络通过分层抽象、模块化结构和更易优化的特性，在实际机器学习任务中表现更优。知识蒸馏等技术也表明，深度和浅层可以互补，但深度结构为大规模复杂任务提供了更高效的解决方案。
