# 2. 算法解析

## 2.1 算法一：暴力解法

**思路：**

- 枚举回文串的中心，然后从中心向两侧扩展以寻找以该中心为最远的回文子串。
- **回文中心有两类：**
  - <font color=amber>奇数长度：</font>以字符为中心，初始左右下标 l = i, r = i。
  - <font color=plum>偶数长度：</font>以字符间隙为中心，初始左右下标 l = i, r = i+1。
- *对每个中心执行相同的扩展过程：* 当 l>=0 && r<n && s[l]==s[r] 时，向外扩展（l--, r++）。扩展结束后，当前回文子串为 s.substr(l+1, r-(l+1))，若长度大于已记录的最大长度则更新答案。
- *<font color=amber>也可以将两类中心合并为一次循环：</font>* 遍历 i=0..2*n-1，令 l = i/2，r = (i+1)/2，这样既能覆盖奇数也能覆盖偶数中心。

**复杂度：**

- <font color=royalblue>时间复杂度 O(n^2)</font>：每个中心最坏扩展到字符串长度；
- <font color=royalblue>空间复杂度 O(1)：</font>只记录若干索引与当前最大子串信息（返回值占额外空间可忽略）。

> **PS: <font color=royalblue>substr</font> 使用说明（兼顾常见语言）：**
> <details open markdown="1">
> 
> - 常见形式： substr(start, length)
>   - start：起始索引（通常 0-based）。
>   - length：要截取的字符数。
>   - 等价区间为 [start, start+length)，即左闭右开。
> - 在扩展法中，若扩展停止时指针为 l,r，则正确的截取方式是 s.substr(l+1, r-(l+1)) —— 起点为 l+1，长度为 r-(l+1)。
> 
</details>

**示例代码：**

```
class Solution
{
public:
    string longestPalindrome(string s)
    {
        int n = s.size();
        int start = 0, max_len = 1;
        for (int i = 0; i < 2 * n - 1; ++i)
        {
            int l = i / 2, r = (i + 1) / 2;
            while (l >= 0 && r < n && s[l] == s[r])
            {
                --l;
                ++r;
            }
            if (r - l - 1 > max_len)
            {
                max_len = r - l - 1;
                start = l + 1;
            }
        }

        return s.substr(start, max_len);
    }
};

```

# 3. Manacher 算法

## 3.1 算法基本思路

- **将原串 s 变换为带分隔符的新串 t**（在两字符间插入特殊字符，以统一奇偶长度），常用做法：
  - <font color=camel>t = "^" + "#" + s[0] + "#" + s[1] + "#" + ... + s[n-1] + "#" + "$"</font>
  - 这样 t 上的任意回文<font color=chocolate>必然是奇长且有明确的中心</font>(2*n+1)。
- 定义数组 p，**<font color=chocolate>p[i] 表示以 t[i] 为中心的回文“半径”</font>（包括中心）**，意味着 t 上的回文区间为 *[i - p[i] + 1, i + p[i] - 1]。*
- <font color=camel>维护当前已知的最右回文区间的中心 C 和右边界 R</font>（通常令 R = C + p[C]）。遍历 i 从左到右：
  - 若 <font color=plum>i < R</font>，则先令 p[i] = min(p[mirror], R - i)，其中 mirror = 2*C - i（i 关于 C 的镜像）。
